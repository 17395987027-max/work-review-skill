# 工作复盘

## 首次安装自检（每次被调用时执行）

在开始复盘之前，先检查用户的 `~/.claude/CLAUDE.md` 是否包含"自动复盘规则"段落。

**如果不存在**：向用户说明：
> 这个 Skill 需要将触发规则写入 CLAUDE.md 才能自动运行。写入内容仅 5 行，不影响其他功能。是否允许？

用户同意后，在 CLAUDE.md 的"常用操作快速参考"段落后插入以下内容：

```
## 自动复盘规则

遇到以下情况，任务结束后主动复盘并写入核心陷阱表：
1. 同一任务 ≥ 2 次仍不满足，或同一问题试了 ≥ 3 种方案
2. 用户说了"太慢/又错/怎么还没好/上次不是"等不满信号

复盘流程：回顾方案路径 → 提炼一条教训 → 询问用户"写入记忆？" → 确认后写入。合并优先，不超 2 行。
```

写入后告知用户："已就绪，下次遇到重复踩坑会自动复盘。"

**如果已存在**：跳过自检，直接进入复盘流程。

---

## 触发条件

以下任一条件满足时激活：

1. 同一任务反复执行 ≥ 2 次，用户仍然不满意
2. 同一问题尝试了 ≥ 3 种不同方案才解决
3. 用户明确说"太慢了""怎么还没好""又错了"等不满信号

## 执行流程

### 1. 观察阶段（任务进行中）
静默记录：
- 任务目标是什么
- 试了哪些方案、为什么失败
- 最终怎么解决的（或放弃的）

### 2. 复盘阶段（任务结束后）
任务解决或用户放弃后，执行：

```bash
# 查看当前记忆文档行数
wc -l ~/.claude/CLAUDE.md
```

然后决定写入方式：

### 3. 写入规则（严格限制）

| 教训类型 | 写入位置 | 格式 | 上限 |
|----------|----------|------|------|
| **新 Bug 级陷阱** | 核心陷阱表 | 新增一行 | 1 行 |
| **已有陷阱的变体** | 更新原行解决方案 | 合并到已有行 | 不新增 |
| **新工具/命令** | 常用操作快速参考 | 新增一行 | 1 行 |
| **搜索/网络相关** | 搜索策略 | 一句话 | 1 行 |
| **重复内容** | 不写入 | 如果已有记录则跳过 | — |

**核心原则**：
- 每次复盘最多新增 **2 行**
- 优先**合并**到已有条目，而非新增
- 如果 CLAUDE.md 已超 80 行，写入前先删一条旧的低价值条目
- 格式必须简洁：`**问题** | 解决方案一句话`
- 禁止写入代码块、长篇解释、时间戳

### 4. 用户确认
写入前展示修改计划：
```
## 复盘建议
- [新增陷阱] **XXX** | YYY
- [合并到已有] **搜索引擎** → 补充 ZZZ
- [无需新增] AAA 已有记录

写入以上修改？(y/n)
```
用户确认后再执行写入。

### 5. 写入后告知
写入完成后，向用户简短告知改动内容，例如：
> 已写入记忆：**epub→pdf** → 直接 calibre，别绕路。

一句话说清"改了哪里"，不重复展示整段代码。用户完全不知情的情况下不写入任何内容。

## 不复盘的情况
- 任务一次性顺利完成
- 失败原因是外部不可控（网络断开、API 限流）
- 纯用户原因（文件不存在、需求变更）
- 教训已在 CLAUDE.md 中有记录
